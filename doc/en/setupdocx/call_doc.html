7.3. call_doc.sh¶

The call_doc.sh implements a wrapper for the generation of the API documentation which is called as subprocess. The curent final default call is:

sphinx-build

The expected input is the raw output from the tool sphinx-apidoc, which in particular includes the appropriate Makefile. The processing by call_doc.sh first prepares the input by adding additional files and an adapted configuration file conf.py. The finished input source directory is finally processed by sphinx.

				exec_call cp -pPr "${BUILDPATH}_build/${_BUILDOUT}/"* "${DOCX_BUILDDIR}/doc/${DOCX_DOCNAME}/"
			fi
		
			# quick-fix for missing custom.css, and ev. others
			if [ -e "${STATIC}" ];then
				exec_call cp -pPr "${STATIC}" "${DOCX_BUILDDIR}/doc/${DOCX_DOCNAME}/"
			fi
		}
		;;

	latexpdf|latexpdfja)
		{
			if [ -e "${BUILDPATH}_build/${_BUILDOUT}" ];then
				exec_call cp -pPr "${BUILDPATH}_build/${_BUILDOUT}/"*.pdf "${DOCX_BUILDDIR}/doc/${DOCX_DOCNAME}/"
			fi
		}
		;;

	epub)
		{
			if [ -e "${BUILDPATH}_build/${_BUILDOUT}" ];then
				exec_call cp -pPr "${BUILDPATH}_build/${_BUILDOUT}/"*.epub "${DOCX_BUILDDIR}/doc/${DOCX_DOCNAME}/"
			fi
		}
		;;

	man)
		{
			if [ -e "${BUILDPATH}_build/${_BUILDOUT}" ];then
				exec_call cp -pPr "${BUILDPATH}_build/${_BUILDOUT}/"* "${DOCX_BUILDDIR}/doc/${DOCX_DOCNAME}/"
			fi
		}
		;;

	*)
		{
			if [ -e "${BUILDPATH}_build/${_BUILDOUT}" ];then

				exec_call cp -pPr "${BUILDPATH}_build/${_BUILDOUT}/"* "${DOCX_BUILDDIR}/doc/${DOCX_DOCNAME}/"
			fi
		}
		;;
			
esac | \
if((_STATE & 6));then
	cat
